Tutorial for Google Pixel Flash Magisk

发表于2022-12-07|更新于2022-12-08|technology
|字数总计:2.2k|阅读时长:7分钟|阅读量:

前言

我写一下在Windows上关于如何将以Google Pixel为主的Android刷入Magisk的方法

我用最无脑但是有点繁琐的方式来教会刷机经验低的小白

所以算是从零开始的刷机教程

请先全部看完,再按步骤开始刷机,当然其他手机也可以参考该方案来进行刷机

强烈建议全部看完再尝试刷机,因为我可能会把一些要点在其它段落陈述出来

但是请记住这句话

玩机有风险,刷机需谨慎,决定权于你,我不负责任

第一步 — — 事前准备

耐心和冷静

有ADB工具和Google USB Driver的可以跳过第二和第三步

有解锁Bootloader的跳过第四步和第五步

第二步 — — 刷机环境的安装和汉化

在电脑上安装Android Studio,软件安装路径任意

如果看不懂英文,那可以通过以下两种方式进行汉化:

1. Android Studio 汉化插件下载地址

下载完成后,然后在

\\Installation path\\Android Studio\\plugins

将插件解压在路径内即可

2. Android Studio 汉化包下载地址

下载完成后,然后在

\\Installation path\\Android Studio\\lib

将汉化包粘贴在此

然后打开Android Studio,找到Help中的Edit Custom VM Options...的选项

修改MyApplication,修改方法如下

换行输入-Duser.language=zh

再换行输入-Duser.region=CN

再重启即可   

其实可以直接下载ADB工具,不过我觉得这个最简单,特别是你不修改第二步SDK Tools的路径

第三步 — — 刷机环境的配置和完善

在Android Studio初始页面内按F打开文件

再按T打开设置

然后找到系统设置中的Android SDK

然后点击其中的SDK Tools

然后勾选Android SDK Platform-Tools和Google USB Driver,然后等待安装完毕

关于Android SDK Platform-Tools和Google USB Driver

如果你嫌麻烦,那也可直接下载Android SDK Platform-Tools和Google USB Driver

以下是关于它们的简短介绍

Android SDK Platform-Tools是调试工具

安装后可以在adb.exe所在路径下用命令行调试Android

当然也可以配置环境变量,然后在任意路径下使用adb

我会在下文介绍如何配置

Google USB Driver是Android解锁Bootloader需要的驱动程序

如果你的手机是其他厂商的,请根据下面的清单寻找符合你手机品牌的驱动程序来解锁Bootloader

  • Acer 宏碁

  • Alcatel one touch 阿尔卡特

  • Asus 华硕

  • Blackberry 黑莓

  • Dell 戴尔

  • Fujitsu 富士通

  • HTC 宏达

  • Huawei 华为

  • Intel 英特尔

  • Kyocera 京瓷

  • Lenovo 联想

  • LGE 乐金

  • Motorola 摩托罗拉

  • MTK 联发科

  • Samsung 三星

  • Sharp 夏普

  • Sony 索尼

  • Toshiba 东芝

  • Xiaomi 小米

  • ZTE 中兴

第四步 — — 刷机环境的设备准备

在手机设置>系统>关于手机>版本号的选项中连点几下即可打开开发者模式

打开开发者选项中的OEM解锁和USB调试

然后用USB连接手机和电脑,在控制面板>设备管理器里更新你的手机驱动

没有USB连接之前和USB连接之后一对比,设备管理器会多出来一个设备,那个就是你的手机

然后鼠标右键点击然后按照提示更新驱动

第五步 — — 解锁Bootloader

第一小节 — 在adb.exe所在路径下使用Windows PowerShell

打开Android SDK Platform-Tools的安装路径

默认的安装路径为

%LocalAppData%\Android\Sdk\platform-tools

直接在文件资源浏览器里输入即可

然后在抵达的文件夹空白处

按住Shift点击鼠标右键,找到PowerShell

然后点击进入Windows PowerShell窗口

第二小节 — 使用Fastboot查错

然后输入以下命令将手机重启至FastBoot模式

adb reboot bootloader

进入Bootloader界面,接着输入以下命令

fastboot devices

来查看USB驱动更新是否成功

如果没有,请通过手机电源按键重启手机,然后重复并自行查错第三步

如果出现设备序列号,则代表你成功更新驱动

第三小节 — 解锁Bootloader

注意! 解锁Bootloader会重置全部手机资料,请记得备份

接下来再在Windows PowerShell窗口输入以下命令解锁手机Bootloader

fastboot flashing unlock

然后手机界面会出现操作界面,接着用音量键找到Unlock the bootloader

最后用电源键确认,确定完成后

在Windows PowerShell窗口里输入fastboot reboot重启手机

重启手机后第三步会被重置,请记得重新开启开发者模式和USB调试

还有一些其他品牌手机的解锁工具方式有些不同,请参考该手机的官方文档

第四小节 — 在adb.exe所在路径下使用Windows PowerShell的替代方案

注意! 本小节只是对第一小节的补充 可看可不看

众所周知,有些步骤可以反过来

比如先打开Windows PowerShell,然后通过

cd C:\\Users\\your name\\AppData\\Local\\Android\\Sdk\\platform-tools

也可以达成第一小节的目标,当然前提是你没有在注册表更改过用户的配置之类

不过,你都懂得改了,那肯定也知道你的platform-tools下载路径在哪里

至于直接下载adb工具的就更不用说了

不过最简单方便方法是配置环境变量

在控制面板>所有控制面板项>用户帐户的更改我的环境变量中

找到Path,点击新建,添加adb所在的路径之后点击确认

然后打开Windows PowerShell,可以直接输入adb和fastboot命令

不过你adb和fastboot所需要的文件还是要亲自到路径下剪贴或复制的

哪怕是用Windows PowerShel剪贴或复制过去的,也是要输入的

第六步 — — 刷入Magisk

第一小节 — 准备前置文件

手机解锁重启之后,在手机设置里找到你的手机的版本号或手机型号

然后最好在电脑上下载相应的手机厂商的镜像文件,在手机下载比较麻烦

这是谷歌官方的—Google Android Phone Images

然后在相应的镜像文件中解压找到boot.img

如果解压一次还是没有找到,可能是因为部分Android ROM文件体积太大

所以厂商会对ROM包进行二次压缩

那就继续在被解压出来的文件继续解压,,终会找到boot.img

接下来在手机下载Magisk Manager

官方下载链接—Magisk Manager

然后安装到手机里,并将解压出的boot.img文件复制到手机内部存储

第二小节 — 前置文件使用

然后打开Magisk Manager

选择安装Magisk

安装方式为”选择并修补一个文件”

点击后会打开文件管理器让你选择文件

那你就选择刚刚移动到你可以找到的路径下的boot.img文件

接着Magisk就会在同路径下生成一个名为”magisk_patched_Name.img”的文件

magisk_patched_Name.img中的Name是随机名,你只要注意前面的”magisk_patched”

然后你需要把magisk_patched_Name.img的文件复制到电脑里

最好和之前的boot.img一起在adb.exe所处于的路径下

达成此步之后

重复之前的,按住Shift点击鼠标右键,找到PowerShell,然后点击进入Windows PowerShell窗口

第三小节 — 使用FastBoot命令

然后还是之前的命令,将手机重启至FastBoot模式

adb reboot bootloader

然后输入以下命令刷入Magisk

fastboot flash boot magisk\_patched\_Name.img

输入完毕后,等待命令行上数值变化结束,

然后在输入以下命令重启手机

fastboot reboot

重启手机后打开Magisk Manager即可发现Magisk安装成功

如果你的手机有些比较特殊的问题,

那你可以然后输入以下命令救砖

fastboot flash boot boot.img

第四小节 — 突发状况处理

当然造成你的手机刷入Magisk失败的原因

可能是部分厂商手机(其实就是Xiaomi啦)设置Android Verified Boot没有通过

Android Verified Boot”是Android启动时验证,简称AVB

比如MIUI会自动将你手机的BootLoader重新锁定

那之前的fastboot flash boot boot.img就解决不了这个问题

你必须要在网络自行下载对应机型的ROM包,解压缩并找到其中的vbmeta.img

然后移动到adb所在路径

然后输入以下命令禁用AVB验证

fastboot --disable-verity --disable-verification flash vbmeta vbmeta.img

然后刷入以下命令重启手机

fastboot reboot

重启后Magisk也会成功安装上
